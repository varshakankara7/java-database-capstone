1. Doctor Persona
Title: View Scheduled Appointments

User Story: As a Doctor, I want to see a list of my daily appointments so that I can prepare for patient consultations.

Acceptance Criteria: * Must display patient name, time, and reason for visit.

Appointments must be sorted chronologically.

The list must update in real-time when an admin adds a booking.

Priority: High

Story Points: 3

Title: Access Patient Medical History

User Story: As a Doctor, I want to view a patient's past prescriptions and diagnoses to provide informed care.

Acceptance Criteria:

Searchable by Patient ID.

Displays a list of past dates with associated notes.

Restricted to authorized medical staff only.

Priority: High

Story Points: 5

2. Admin Persona
Title: Register New Doctor

User Story: As an Admin, I want to add new doctor profiles to the system so they can begin taking appointments.

Acceptance Criteria:

Admin can input name, specialization, and license number.

System validates that the license number is unique.

Successful submission triggers an email notification to the doctor.

Priority: Medium

Story Points: 3

3. Patient Persona
Title: Book Appointment

User Story: As a Patient, I want to book a slot with a specific specialist so I can receive medical advice.

Acceptance Criteria:

Users can filter doctors by specialization.

The system only shows available time slots.

Confirmation message is displayed after successful booking.

Priority: High

Story Points: 8

Key Improvements Made
Detailed Acceptance Criteria: Clearly defines when a feature is considered "complete."

Priority Levels: Helps stakeholders understand which features must be built first (e.g., "Book Appointment" is High, "Update Profile Picture" might be Low).

Story Points: Provides a metric for development effort, helping in sprint planning.
